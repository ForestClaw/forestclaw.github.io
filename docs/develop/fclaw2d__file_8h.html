Detailed Description

Routines for parallel IO for fclaw2d_domain_t and generic data.

Principle: All files are associated with a single domain, i.e. one can write only one domain to one file.

Open a file for reading and read the stored domain.

The user string is broadcasted to all ranks after reading. The file must exist and be at least of the size of the file header.

This is a collective function. If the file has wrong metadata the function reports the error using fclaw_errorf, collectively closes the file and deallocates the file context. In this case the function returns NULL on all ranks. The wrong file format or a wrong file header causes FCLAW2D_FILE_ERR_FORMAT as errcode.

After calling this function the user can continue reading the opened file by calling fclaw2d_file_read_block, fclaw2d_file_read_array and potentially other functions operating on fclaw2d_file_context_t that may be implemented on the top of the current fclaw2d_file routines.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in]filenameThe path to the file that is opened.
[out]user_stringAt least FCLAW2D_FILE_USER_STRING_BYTES bytes. The user string is written to the passed array including padding spaces and a trailing NUL-termination.
[in]mpicommMPI communicator that is used to read the file and is used for potential other reading operations of MPI communicator dependent objects.
[in]partitionA sc_array of the size of number of MPI ranks + 1 with sizeof (p4est_gloidx_t) as element size or NULL. The sc_array must contain a valid partition, i.e. the first element is 0, the array is non-decreasing and the last array entry equals the number of global patches of the domain in the file pointed to by filename. If any of these conditions is violated, the function returns NULL and set errcode to FCLAW2D_FILE_ERR_PART. The user can pass NULL for using the uniform partition with respect to the quadrant count. In both cases the respective partition, either read or computed, is used for the parallel I/O operations and stored in the returned domain. The user can use fclaw2d_file_read_partition to read a partition array from file.
[out]domainNewly allocated domain that is read from the file.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Newly allocated context to continue reading and eventually closing the file. NULL in case of error.

Create and open a file that is associated with the given domain structure.

This is a collective function call that overwrites the file if it exists already. This function writes a header with metadata of the underlying p4est of the passed domain to the file. In addition, the passed domain is written to the file.

The opened file can be used to write to the file using the functions fclaw2d_file_write_block, fclaw2d_file_write_array and potentially other functions operating on fclaw2d_file_context_t that may be implemented on the top of the current fclaw2d_file routines.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in]filenamePath to parallel file that is to be created.
[in]user_stringA NUL-terminated user string that is written to the file header having FCLAW2D_FILE_USER_STRING_BYTES bytes including the NUL-termination. Shorter user strings are padded by spaces in the file header. Too long user strings result in an error with the error code FCLAW2D_FILE_ERR_IN_DATA.
[in]domainThe underlying p4est is used for the metadata of the the created file and the domain is written to the file.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Newly allocated context to continue writing and eventually closing the file. NULL in case of error.

Read per-patch data from a parallel output file.

This is a collective function. The function closes and deallocates the file context and returns NULL if the bytes the user wants to read exceed the given file and/or the element size of the array given by patch_data->elem_size does not coincide with the element size according to the array metadata given in the file.

The data is read in parallel using the partition of the domain (and the underlying p4est) that is associated with the passed file context in the sense that the file context was created using the respective domain. The data size per patch must be fixed.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in]fcContext previously created by fclaw2d_file_open_read. It keeps track of the data sets read one after another.
[out]user_stringAt least FCLAW2D_FILE_USER_STRING_BYTES bytes. The user string is written to the passed array including padding spaces and a trailing NUL-termination.
[in]patch_sizeThe number of bytes per patch. This number must coincide with patch_data->elem_size.
[in,out]patch_dataAn array of the length number of local patches with the element size equal to sizeof (sc_array_t). This means each element of patch_data must be a sc_array. These sc_arrays must have an element size of patch_size. This memory layout enables the user to read by indirect addressing, i.e. to read into non-contigous patch data. The patch data is read according to the Morton order of the patches. patch_size must coincide with the section data size in the file. patch_data == NULL means that the data is skipped and the internal file pointer is incremented. In the case of skipping patch_size is still checked using the corresponding value read from the file. The data is read using the partition of patches given by the domain that is associated to fc.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Return a pointer to input context or NULL in case of errors that does not abort the program. In case of error the file is tried to close and fc is freed.

Read a serial data block from an opened file.

This is a collective function.

This function requires an opened file context.

The passed block_size is compared to the block size stored in the file. If the values do not equal each other, the function reports details, closes and deallocates the file context. The return value in this case is NULL. If the data block section header information is not matching the passed parameters the function sets FCLAW2D_FILE_ERR_FORMAT for errcode.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in]fcContext previously created by fclaw2d_file_open_read. It keeps track of the data sets read one after another.
[out]user_stringAt least FCLAW2D_FILE_USER_STRING_BYTES bytes. The user string is written to the passed array including padding spaces and a trailing NUL-termination.
[in]block_sizeThe size of the header that is read.
[in,out]block_datablock_size allocated bytes in an sc_array with one element and block_size as element size. This data will be filled with the block data from the file. If this is NULL it means that the current header block is skipped and the internal file pointer of the file context is set to the next data section. If the current data section is not a data block, the file is closed and the file context is deallocated. Furthermore, in this case the function returns NULL and sets errcode to FCLAW2D_FILE_ERR_FORMAT. In case of skipping the data block section block_size needs also to coincide with the data block size given in the file.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Return a pointer to input context or NULL in case of errors that does not abort the program. In case of error the file is tried to close and fc is freed.

Read a partition array from file.

This function reads the partition array, i.e. the global first patch array from a partition file that was written using fclaw2d_file_write_partition. The read partition can be passed to fclaw2d_file_open_read.

This function guarantees that on successful output the read partition is valid in the sense that the partition array has 0 as first entry and is non-decreasing.

The function and all its parameters are collective.

This function does not abort on MPI I/O errors but returns -1. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in]filenameThe path to the partition file.
[out]user_stringAt least FCLAW2D_FILE_USER_STRING_BYTES bytes. The user string is written to the passed array including padding spaces and a trailing NUL-termination.
[in]mpicommThe MPI communicator is required to synchronize the errcode and the output array partition. It is important to notice that partition is only available on MPI ranks that are part of the passed mpicomm. Therefore, mpicomm should coincide with the mpicomm parameter of fclaw2d_file_open_read to ensure that the partition parameter of fclaw2d_file_open_read is collectivly available.
[out]partitionA sc_array with element size equals to sizeof (p4est_gloidx_t). On successful output the array is filled with the read partition. In case of an error, the function does not guarantee that partition stays untouched but it is guaranteed that the user is not required to call sc_array_reset on partition.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
0 for a successful read of the partition file. -1 in case of an error.

Write per-patch data to a parallel output file.

This is a collective function. This function writes the per-patch data with respect the domain that was passed to the fclaw2d_file_open_write for opening the file.

The per-patch data is written in parallel according to the partition of the domain and the underlying p4est, respectively. The data size per patch must be fixed.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters
[in,out]fcContext previously created by fclaw2d_file_open_write. It keeps track of the data sets written one after another.
[in]user_stringA NUL-terminated user string that is written to the section header having FCLAW2D_FILE_USER_STRING_BYTES bytes including the NUL-termination. Shorter user strings are padded by spaces in the section header. Too long user strings result in an error with the error code FCLAW2D_FILE_ERR_IN_DATA.
[in]patch_sizeThe number of bytes per patch. This number must coincide with patch_data->elem_size.
[in]patch_dataAn array of the length number of local patches with the element size equal to sizeof (sc_array_t). This means each element of patch_data must be a sc_array. These sc_arrays must have an element size of patch_size and store the actual patch data. This memory layout enables the user to write by indirect addressing, i.e. to write non-contigous patch data. The patch data is expected to be stored according to the Morton order of the patches. For patch_size == 0 the function writes an empty array. The section header and the padding is still written. In this case the function passes successfully and errcode is set to FCLAW2D_FILE_ERR_SUCCESS.
[out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Return a pointer to input context or NULL in case of errors that does not abort the program. In case of error the file is tried to close and fc is freed.

This is a collective function. This function writes a serial data block to the opened file.

The number of block bytes must be less or equal FCLAW2D_FILE_MAX_BLOCK_SIZE.

This function does not abort on MPI I/O errors but returns NULL. Without MPI I/O the function may abort on file system dependent errors.

Parameters

[in]user_stringA NUL-terminated user string that is written to the section header having FCLAW2D_FILE_USER_STRING_BYTES bytes including the NUL-termination. [out]errcodeAn errcode that can be interpreted by fclaw2d_file_error_string.
Returns
Return a pointer to input context or NULL in case of errors that does not abort the program. In case of error the file is tried to close and fc is freed.
